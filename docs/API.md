# API.md Template

## 문서 목적
소스 코드 정적 분석을 통해 확인 가능한 API 엔드포인트 정보를 기록한다.

## 작성 원칙

- 확인되지 않은 endpoint는 쓰지 않는다.
- 응답/예외/권한 정보는 코드나 메타데이터에서 확인된 값만 적는다.
- 불확실한 항목은 `UNKNOWN` 또는 `needs_review`로 표기한다.
