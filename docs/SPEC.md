# SPEC.md Template

## 문서 목적
변경 파일(diff)과 정적 분석 결과를 기준으로 기능 단위의 변경 사항을 기록한다.

## 작성 원칙

- 변경 사실은 Git diff와 소스 코드 evidence로만 정리한다.
- 영향 범위는 추정하지 말고 확인 가능한 연결만 적는다.
- 서술형 summary가 필요하면 팩트 섹션과 분리한다.
